## Phan 1: Direct LLM Calling

### Buoc 2: Doc va hieu code

1. LLM duoc khoi tao nhu the nao?

LLM duoc khoi tao thong qua ham `get_llm()` trong `common/llm.py`. Ham nay tra ve mot doi tuong `ChatOpenAI`, cau hinh de goi OpenRouter bang `OPENROUTER_API_KEY`, `OPENROUTER_MODEL`, `openai_api_base`, `temperature` va `max_tokens`.

2. Message duoc gui den LLM co cau truc gi?

Message la mot list gom 2 loai message:

- `SystemMessage`: dat vai tro va huong dan cho LLM, vi du yeu cau tra loi nhu mot chuyen gia phap ly.
- `HumanMessage`: chua cau hoi thuc te cua nguoi dung.

3. Tai sao can co `SystemMessage` va `HumanMessage`?

`SystemMessage` giup dinh huong cach LLM tra loi, bao gom vai tro, phong cach va gioi han cau tra loi. `HumanMessage` la noi dung cau hoi can xu ly. Tach hai loai message nay giup code ro rang hon va giup LLM hieu dau la instruction, dau la input cua user.

### Buoc 2: Phan tich code

1. Ham `@tool` decorator duoc dung o dau?

`@tool` duoc dung truoc cac ham `search_legal_database` va `calculate_damages` trong `stages/stage_2_rag_tools/main.py`. Decorator nay bien cac ham Python thanh tools ma LLM co the goi.

2. `LEGAL_KNOWLEDGE` duoc cau truc nhu the nao?

`LEGAL_KNOWLEDGE` la mot list cac dictionary. Moi dictionary dai dien cho mot nguon kien thuc phap ly va co cac truong:

- `id`: ma dinh danh cua entry.
- `keywords`: danh sach tu khoa de search matching.
- `text`: noi dung kien thuc phap ly tra ve khi tim thay entry phu hop.

3. LLM duoc bind voi tools ra sao?

LLM duoc bind voi tools bang dong:

```python
llm_with_tools = llm.bind_tools(TOOLS)
```

Ban dau `TOOLS` gom `search_legal_database` va `calculate_damages`. Sau Bai tap 2.2, `TOOLS` gom them `check_statute_of_limitations`. Sau khi bind, LLM co the quyet dinh goi tool nao va truyen arguments nao vao tool.

### Buoc 3: Doc code

1. `create_react_agent()` nam o dau?

Trong `stages/stage_3_single_agent/main.py`, ham `create_react_agent()` duoc import trong `main()` va duoc goi de tao graph agent:

```python
graph = create_react_agent(model=llm, tools=TOOLS, prompt=SYSTEM_PROMPT, debug=True)
```

2. So sanh voi Stage 2

Stage 2 phai tu viet manual tool loop: LLM goi tool, code execute tool, append `ToolMessage`, roi goi LLM lan nua.

Stage 3 dung ReAct agent nen agent tu quan ly vong lap Think -> Act -> Observe. Code khong can tu viet tung buoc tool-call loop nhu Stage 2.

3. `agent_executor.invoke()` hoac `graph.astream()` hoat dong nhu the nao?

Trong code hien tai, Stage 3 dung `graph.astream(inputs, stream_mode="updates")` de stream tung buoc cua agent. Moi update cho biet agent dang goi tool nao, ket qua observe la gi, va final answer khi hoan thanh.

## Phan 4: Multi-Agent In-Process

### Buoc 1: Chay demo Stage 4

Da chay lenh:

```bash
uv run python stages/stage_4_milti_agent/main.py
```

Lan chay dau bi loi OpenRouter credit khi `max_tokens=500`. Da giam default `OPENROUTER_MAX_TOKENS` trong `common/llm.py` xuong `300`, sau do Stage 4 chay duoc qua cac node:

- `analyze_law`
- `check_routing`
- `call_tax_specialist`
- `call_compliance_specialist`
- `call_privacy_specialist`

Lan chay co privacy branch da xac nhan router tra:

```text
needs_tax=True, needs_compliance=True, needs_privacy=True
```

Va 3 specialist agents deu chay song song. Tuy nhien API bi loi o buoc `aggregate` do OpenRouter credit con lai chi afford 298 token trong khi request 300. Sau khi giam tiep default xuong `250`, account bao `Insufficient credits`, nen khong the chay full final answer them nua.

### Buoc 2: Phan tich kien truc

1. `class LegalState(TypedDict)` la shared state cua graph.

State gom cac field chinh:

- `question`: cau hoi goc.
- `law_analysis`: ket qua phan tich phap ly tong quat.
- `needs_tax`, `needs_compliance`, `needs_privacy`: flags routing.
- `tax_result`, `compliance_result`, `privacy_result`: ket qua cua cac specialist agents.
- `final_answer`: cau tra loi tong hop cuoi cung.

2. Cac agent functions:

- `analyze_law`: lead attorney phan tich phap ly tong quat.
- `check_routing`: quyet dinh can goi specialist nao.
- `call_tax_specialist`: tax specialist.
- `call_compliance_specialist`: compliance specialist.
- `call_privacy_specialist`: privacy/GDPR specialist moi them.
- `aggregate`: tong hop tat ca ket qua.

3. `Send()` API duoc dung o dau?

`Send()` duoc dung trong ham `route_to_specialists`. Ham nay tao cac `Send(...)` objects de dispatch song song den cac node specialist:

```python
Send("call_tax_specialist", state)
Send("call_compliance_specialist", state)
Send("call_privacy_specialist", state)
```

4. `graph.add_node()` va `graph.add_edge()`

`create_graph()` tao `StateGraph(LegalState)`, add cac nodes, sau do noi flow:

```text
analyze_law -> check_routing -> [tax + compliance + privacy] -> aggregate -> END
```
